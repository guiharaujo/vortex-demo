# Vórtex — Supply Chain Planning Dashboard

A **Streamlit-based supply chain planning system** that pulls live data from an ERP (SAP B1 / SQL Server) and delivers a unified view for PCP and S&OP teams. Built for industrial manufacturers managing finished goods, raw materials, production orders, and purchase workflows.

Runs locally or on a private server, accessed by the planning team via internal network (e.g. Tailscale VPN).

---

## Features

- **Production capacity cascade** — ABC-ranked simulation of how many units can be built given current component stock; bottleneck component highlighted per finished good
- **Stockout calendar** — 52-week chronological simulation with actual shipment dates distributed per sales order; statuses: OK / ATTENTION / STOCKOUT / INCOMING
- **MRP (Material Requirements Planning)** — net purchase needs per component with urgency scoring (🔴 TODAY / 🟡 7 days / 🟢 OK); separate tab for resale items
- **S&OP scenario editor** — editable % adjustments per SKU with 3 scenarios (pessimistic / base / optimistic); persisted to local Excel
- **Historical sales** — 3-month closed periods + current month YTD, aligned with order-backed invoices only (excludes demos, internal transfers, future-sale first NF)
- **BOM explosion** — multi-level bill of materials with stock coverage per node
- **Purchase workflow** — open purchase requisitions linked to PO drafts with approval status (approved / pending / rejected)
- **Role-based access** — admin (data refresh, exports) and read-only (planner/buyer) profiles; HMAC-signed URL token persists across page navigation
- **Adaptive CSS** — works on both light and dark Streamlit themes; Inter + JetBrains Mono typography

### Data flow

```
ERP (SAP B1 / SQL Server)
        │
        ▼  pyodbc
atualizar_dados.py  ──── capacidade_producao.py (queries + calculations)
        │
        ▼  atomic write (tmp → rename)
dados/dados_sap.xlsx  (20+ sheets)
        │
        ▼  pd.read_excel  @st.cache_data(ttl=3600)
Streamlit pages
```

## Calculation engine highlights

### ABC curve
Cumulative Consumo3m-based ranking: A (top 80%), B (80–95%), C (95–100%). Applied independently to finished goods and raw materials.

### Production capacity cascade
Priority order: A items → B → C; within class, sorted by urgency (carteira / days of coverage). For each SKU, expands BOM recursively and finds the bottleneck component (lowest `floor(stock / qty_per_unit)`). Resale items bypass BOM — capacity = `stock + incoming POs`.

### Stockout calendar (chronological simulation)
```
For each week (1 → 52):
  outflow = max(weekly_average, sum of orders with ship_date in this week)
  stock   = stock - outflow + arrivals_this_week

  if stock >= 0 and arrivals > 0 → INCOMING
  if stock >= 2 weeks buffer      → OK
  if stock >= 0 but buffer < 2w   → ATTENTION
  if stock < 0 and future arrivals cover deficit → ATTENTION (delayed, not lost)
  if stock < 0 and nothing covers → STOCKOUT
```
Shipment dates from individual sales order lines. Arrivals from PO `ShipDate` and production order `DueDate`. Late/null dates fall to current week.

### MRP
`Net need = max(backlog, avg_day × horizon) - stock - open_POs`
Urgency: 🔴 need ≤ today, 🟡 need ≤ 7 days, 🟢 OK.

## Setup

### 1. Clone and install

```bash
git clone https://github.com/guiharaujo/vortex-demo.git
cd vortex-demo
pip install -r requirements.txt
```

### 2. Environment variables

```bash
cp .env.example .env
# Edit .env with your ERP connection details and auth secret
```

Or set as Windows System Environment Variables (required if running via Task Scheduler).

### 3. Run ETL to generate data

```bash
python atualizar_dados.py
```

### 4. Start the app

```bash
streamlit run Home.py --server.port 8501
```

For internal network access:
```bash
streamlit run Home.py --server.address 0.0.0.0 --server.port 8501 --server.headless true
```

### 5. Auto-start on Windows (Task Scheduler)

Create a task that runs `start_vortex.bat` at system startup with "Run whether user is logged on or not" + highest privileges.

## ERP Compatibility

Built against **SAP B1 (SQL Server backend)** using standard B1 tables:

| Table | Usage |
|---|---|
| `OITM / OITW` | Item master + warehouse stock |
| `ITT1` | Bill of materials |
| `OINV / INV1` | Sales invoices |
| `ORDR / RDR1` | Sales orders (backlog) |
| `OPOR / POR1` | Purchase orders |
| `OWOR` | Production orders |
| `OIGE / IGE1` | Goods issue (production consumption) |
| `OPQR / PQR1` | Purchase requisitions |
| `ODRF / DRF1` | PO drafts |
| `OINM` | Inventory movements |

The query layer (`capacidade_producao.py`) can be adapted to other SQL Server-based ERPs by remapping table names.
